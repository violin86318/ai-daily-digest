### 3. Agents Aren't Coworkers：把它们嵌入你的软件，而不是当成同事

**来源**：[Feldera Blog](https://www.feldera.com/blog/ai-agents-arent-coworkers-embed-them-in-your-software) | HackerNews

Feldera 提出一个反直觉观点：当今的 Agent 设计思路错了——不应该把 AI 当成需要"管理"的同事，而应该把软件改造成适合 Agent 运行的形式，让交互变成"背景化"的。

**核心论点：**
Mark Weiser 1991年的"21世纪计算机"愿景是让技术"消失"——融入生活不引人注意。但今天的 Copilot/Chatbot 恰恰相反：它们像人，需要轮询讨论、解释自己、要求确认。认知负担很高。

**解决方案——三个 Agentic 软件设计模式：**
1. **CLI 接口**：节省 token，减少对话噪音
2. **声明式配置（Spec）**：描述目标状态而非步骤
3. **Reconciliation Loop**（对账循环）：声明目标状态，系统持续向其收敛

**独特洞见——CDC（变更数据捕获）：**
数据库可以主动推送变更事件（插入/更新/删除），而不是让 Agent 反复轮询快照。Agent 收到精确的事件通知，而非昂贵的全量查询。Feldera 作为增量查询引擎，这个模型是其天然优势。

> 📌 **一句话**：不是让 Agent 更聪明，而是让软件更适合 Agent 运行——CDC + Reconciliation Loop = "机器的平静技术"。
