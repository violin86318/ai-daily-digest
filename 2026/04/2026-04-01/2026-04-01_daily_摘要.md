## 🔐 安全

### FreeBSD 远程内核 RCE（CVE-2026-4747）
MADBugs 团队披露了一个严重的 FreeBSD 漏洞：在 `rpcsec_gss` 模块中，`svc_rpc_gss_validate()` 函数将 RPC 凭证体复制到一个 128 字节的栈缓冲区时，**未对 `oa_length` 做边界检查**。攻击者通过构造超长凭证，可覆盖栈上的返回地址，在 NFS 服务器上实现远程代码执行（root shell）。受影响版本：FreeBSD 13.5 <p11、14.3 <p10、14.4 <p1、15.0 <p5。该漏洞由 AI 辅助发现。

```c
// Bug: No bounds check on oa_length!
// After 32 bytes of header, only 96 bytes remain in rpchdr.
// If oa_length > 96, this overflows past rpchdr into:
// local variables → saved callee-saved registers → return address
memcpy((caddr_t)buf, oa->oa_base, oa->oa_length);
```

> [CVE-2026-4747 Write-up (GitHub)](https://github.com/califio/publications/blob/main/MADBugs/CVE-2026-4747/write-up.md)
