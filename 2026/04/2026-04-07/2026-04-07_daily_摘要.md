### Hippo Memory：给 AI Agent 装上"遗忘机制"

GitHub: kitfunso/hippo-memory

生物启发的 AI Agent 记忆系统，核心洞察是：**好记忆的关键不是记住更多，而是知道该遗忘什么**。

核心特性：
- **衰减机制（Decay）**：常用记忆强化，噪音自然消退，类比人类记忆的突触可塑性
- **混合搜索**：BM25 + 向量嵌入（可选 @xenova/transformers）
- **零运行时依赖**，SQLite + Markdown 双存储，可 Git 追踪
- **跨工具共享**：Import from ChatGPT、Claude（CLAUDE.md）、Cursor（.cursorrules），实现跨工具记忆迁移
- **Session Handoff**：会话之间显式传递状态和下一步行动，新会话无需考古聊天记录
- **可解释召回**（`--why`）：展示 BM25/嵌入贡献权重、来源层级
- 实测：50 任务序列中，使用 Hippo 的 agent 陷阱率从 78% 降至 14%

```bash
npm install -g hippo-memory
hippo init
hippo remember "FRED cache silently dropped the tips_10y series" --tag error
hippo recall "data pipeline issues" --budget 2000
```

**关键词：** AI Agent 记忆、遗忘机制、跨工具、hippo-memory

### TermHub：让 AI Agent 完全控制终端

GitHub: duo121/termhub | npm: @duo121/termhub

开源的原生终端控制网关，为 AI Agent 设计的闭环：

1. AI 检查打开了哪些终端会话
2. AI 按需打开窗口或标签页
3. AI 启动或指向 Codex 会话
4. AI 发送任务并只捕获**执行后的增量输出**返回给用户

支持 macOS（iTerm2 / Terminal）和 Windows（Windows Terminal / CMD），提供完整的 SDK (`createTermhubClient`)。

```bash
npm install -g @duo121/termhub
termhub list
termhub send --text "echo hello" --await-output
```

**关键词：** 终端控制、AI Agent、iTerm2、termhub

### mdarena：让你的 CLAUDE.md 和真 PR 同场竞技

GitHub: HudsonGri/mdarena

Research 发现 CLAUDE.md 文件往往**降低 Agent 成功率并增加 20%+ token 消耗**——但这些都是盲测。mdarena 让你自己衡量你的 CLAUDE.md 到底有没有用。

工作流程：
- `mdarena mine owner/repo --limit 50` 从真实代码库挖掘已合并的 PR 作为测试集
- `mdarena run -c claude_v1.md -c claude_v2.md` 多版本 CLAUDE.md 同场对比
- `mdarena report` 统计显著性检验（配对 t 检验）

关键发现（大型生产 Monorepo，20 PRs，Claude Opus 4.6）：
- 现有 CLAUDE.md 提升测试通过率约 **27%**
- 将所有分目录规则合并为一个文件，效果**不比裸跑差**（引入噪音）
- 正确答案：**在正确时机给 Agent 提供正确上下文**，而不是堆砌内容

支持 SWE-bench 导入、`--detect-tests` 自动从 CI 配置推断测试命令。

**关键词：** CLAUDE.md 评估、SWE-bench、Agent 基准测试、mdarena
