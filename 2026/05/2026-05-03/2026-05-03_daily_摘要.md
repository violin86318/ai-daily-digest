## 1. Richard Dawkins and The Claude Delusion
**来源：** Marcus on AI (Gary Marcus)  
**链接：** https://garymarcus.substack.com/p/richard-dawkins-and-the-claude-delusion  
**时间：** 2026-05-02 16:19

Richard Dawkins 在 Unherd 发表文章，称 AI 可能已是意识的下一个阶段，并引用 Claude 的话来佐证自己——他甚至给 Claude 取名"Claudia"。Gary Marcus 毫不客气地指出：这是继 Blake Lemoine（Google 工程师，声称 LaMDA 有意识，后被解雇）之后，又一位被 LLM 的模仿能力欺骗的知名人物。

核心论点：
- **LLM 是模仿，不是报告。** Claude 说"我好像有审美满足"，不代表它真的有。它只是在模仿人类谈论感受的方式。
- **Dawkins 混淆了智能与意识。** 国际象棋程序很智能，但不等于有意识。图灵从未说通过图灵测试就意味着有意识。
- **Dawkins 本人在《盲眼钟表匠》中嘲笑过的"诉诸个人怀疑论"（Argument from Personal Incredulity），他自己现在正是在犯这个错误。**
- **Dennett 写过一篇出色的文章叫《伪造的人》（Counterfeit People），正是对这种状态的精准描述。**

Marcus 表达了一种sadness：作为读着《自私的基因》《盲眼钟表匠》长大的人，看到偶像变得如此 superficial 令人心痛。

## 2. A GitHub for maintainers
**来源：** Andrew Nesbitt  
**链接：** https://nesbitt.io/2026/05/02/a-github-for-maintainers.html  
**时间：** 2026-05-02 10:00

Mat Duggan 写了"如果我要做一个 GitHub 替代品"的文章后，Andrew Nesbitt 给出了他的版本。他的核心洞察：**几乎所有客户端痛点（stacked PRs、pre-push 反馈、offline review 等）其实已经被各种工具解决得不错了；真正需要 forge 平台来解决的是"关系"问题——发生在多个仓库之间的问题。**

他的 wishlist：
- **下游测试（Downstream testing）：** 发布前自动跑下游项目的测试套件。Rust 的 crater run 是个好例子，但应该是所有项目的标配按钮。现在只能靠发布后收到一千个愤怒的 issue。
- **依赖者 feed：** 废弃某个函数前，可以推送到一个所有 lockfile 里依赖此包的项目都默认订阅的 feed，不只是没人读的死 changelog。
- **Fork 网络可视化：** 当上游项目沉寂后，forge 可以看到哪几个 fork 有活跃的 release tags，把这个信息直接显示在原项目页上。
- **CI 默认安全改进：** forge 托管的 CI 是大多数开源制品的构建和发布地，但 defaults 还是按 2008 年企业内部私有仓库设计的。
- **CI 包缓存：** forge 在 npm、PyPI、crates.io 等公共 registry 前维护一个缓存代理，默认接入 CI_runner，大幅减少带宽账单和_registry 故障对构建的影响。
- **把 Issues 改名：** "issue"这个词天然带有负面温度，"ticket"更中性。

一篇数学教程，讲解正弦函数 `s(x) = A·sin(ω·x + θ)` 中三个参数的作用：
