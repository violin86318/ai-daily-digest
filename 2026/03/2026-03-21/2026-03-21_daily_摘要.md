## 🔬 Attention Residuals：取代残差连接的新型 Transformer 架构

MoonshotAI（Kimi 团队）发布 **Attention Residuals（AttnRes）**，一种用注意力机制替代标准残差连接的 Transformer 改进方案。

**核心思想：** 标准残差连接以固定权重均匀累积所有层的输出，当模型加深时出现"PreNorm 稀释"问题。AttnRes 让每层通过一个学习到的伪查询，自适应地选择性地聚合早期层的表示：

$$h_l = \sum_{i=0}^{l-1} \alpha_{i \to l} \cdot v_i$$

其中注意力权重由单个伪查询 $w_l$ 计算得到，实现了内容感知的深度聚合。

**Block AttnRes 优化：** 全注意力的显存需求为 O(Ld)，Block 版本将层分组，只在块边界做注意力，O(Nd) 显存即可获得大部分收益。

**benchmark 结果（与 Baseline 对比）：**
| 任务 | Baseline | AttnRes | 提升 |
|------|---------|---------|------|
| GPQA-Diamond | 36.9 | **44.4** | +7.5 |
| HumanEval | 59.1 | **62.2** | +3.1 |
| MMLU | 73.5 | **74.6** | +1.1 |
| Math | 53.5 | **57.1** | +3.6 |

**结论：** Block AttnRes 能以 1.25x 更少计算量达到相同 loss，且在多步推理（+7.5）和代码生成（+3.1）上提升最显著。

## 🧠 LLM Circuit Finder：无训练即可提升推理能力 17%

通过复制特定 Transformer 层，在不改变权重的情况下，将逻辑推理能力从 0.22 提升至 **0.76（BBH 基准）**，提升幅度达 245%。

**方法：** 基于 David Ng 的 RYS 方法，发现不同模型的"推理回路"位于不同层级：
- **Devstral-24B**（40层）：推理回路在第 12-14 层
- **Qwen2.5-32B**（64层）：推理回路在第 7-9 层

只需复制正确的 3 层块，让隐状态路由通过同一回路两次，模型就能"更认真地思考"。

**不同复制模式产生不同认知画像：**
- 双次通过（13-16）：数学专家模式
- 三次通过（13-16）：情商 EQ 专家模式
- 四次通过（13-16）：EQ 模式，数学中性

全部在 2 张 AMD 消费级 GPU（RX 7900 XT + RX 6950 XT）上一晚上完成。

⚠️ 代价：数学和因果推理提升，但指令遵循和代码生成略有下降（-3%）。

## ⚡ 从 Rust+WASM 切换到 TypeScript：解析器提速 3 倍

OpenUI 分享了一个反直觉的性能调优案例：将 Rust WASM 解析器重写为 TypeScript 后，全流解析速度提升 **3.3x**。

**教训：** WASM 的每次调用都要承担固定的跨边界开销（字符串复制进 WASM 线性内存 → Rust 解析 → JSON 序列化 → 复制出 WASM → V8 JSON.parse），而非 Rust 本身的速度不足。

加上流式增量缓存优化（已完成语句永不重新解析），最终 Rust WASM 方案下 950 字符仪表盘解析需 257µs，纯 TypeScript 只需 55µs。

**关键洞察：** 在流式 LLM 输出的场景中，O(N²) 的重复解析才是真正的性能瓶颈，而非解析算法本身的执行速度。
