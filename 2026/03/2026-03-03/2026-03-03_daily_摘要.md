## 1. 🔥 美国政府全面封杀Anthropic

**来源**: Reuters, NYT, TechCrunch  
**时间**: 2026年3月2-3日

### 核心事件

特朗普政府下令**所有联邦机构停止使用Anthropic产品**，这是美国政府对AI公司最严厉的制裁之一：

- **美国财政部**：宣布终止所有Anthropic产品使用，包括Claude
- **国务院**：将其内部聊天机器人StateChat从Anthropic切换至**OpenAI GPT-4.1**
- **联邦住房金融局**：终止与Fannie Mae和Freddie Mac的所有Anthropic产品使用
- **国防部**：将Anthropic列为**供应链风险**，五角大楼宣布2亿美元AI合同谈判破裂

### 背后原因

据NYT报道，导火索是Anthropic拒绝满足政府关于**AI安全护栏**的要求。Anthropic以安全顾虑为由拒绝向国防部提供某些技术，导致双方谈判破裂。

TechCrunch评论称这是Anthropic自己设的陷阱——作为倡导AI安全的公司，它既承诺负责任地自我管理，又试图拒绝政府监管，最终陷入两难。

### 市场影响

OpenAI迅速抓住机会，宣布与国防部签订**机密网络部署协议**。这标志着美国政府AI供应商格局的重大转变——从多极竞争走向OpenAI主导。

## 2. 🧠 Karpathy年度回顾：2025年LLM六大范式转变

**来源**: karpathy.bearblog.dev  
**时间**: 2025年12月20日发布

Andrej Karpathy发表**《2025 LLM Year in Review》**，总结了改变行业格局的六大范式转变：

### 1. RLVR（可验证奖励的强化学习）

2025年最大的技术突破。传统LLM训练三步曲（预训练→SFT→RLHF）新增第四阶段：**RLVR**。

- 通过数学/代码等可验证环境训练LLM
- LLM自发发展出类似"推理"的策略——将问题分解为中间计算步骤
- DeepSeek R1论文展示了典型案例
- 带来全新的**测试时计算扩展**能力——通过延长推理时间提升性能

Karpathy指出，这改变了算力分配格局——原本用于预训练的算力被RLVR吞噬。

### 2. Ghosts vs Animals（幽灵vs动物）

2025年，行业开始**直观理解LLM智能的"形状"**。

- 我们不是"进化/培育动物"，而是在"召唤幽灵"
- LLM的神经网络架构、训练数据、算法、优化压力都与生物大脑截然不同
- 它们在智能空间中产生了完全不同的实体

Karpathy用**锯齿状性能曲线**描述这种特性：LLM在可验证领域"尖峰"般突出，同时又是困惑的小学生—— genius polymath和confused grade schooler并存。

### 3. Cursor / LLM应用新层

2025年，Cursor揭示了**LLM应用的全新层**——"Cursor for X"概念兴起。

- **上下文工程**：为特定垂直领域打包和组织LLM调用
- **DAG编排**：将多个LLM调用串成复杂DAG，平衡性能与成本
- **领域特定GUI**：为人机协作提供专用界面
- **自主性滑块**：让用户控制AI自主程度

Karpathy认为LLM实验室会培养出"通用大学生"，但LLM应用会将它们组织成特定垂直领域的"专业团队"。

### 4. Claude Code / 活在电脑里的AI

Claude Code (CC) 首次展示了**真正的LLM Agent**——以循环方式串联工具使用和推理进行扩展问题解决。

- **本地运行**：在用户的私人环境和数据上运行
- **打破云优先思维**：OpenAI早期将Codex/Agent部署在云端容器中，Anthropic选择直接在localhost运行
- **全新交互范式**：AI不是"像Google一样访问的网站"，而是"住在电脑里的小精灵/幽灵"

### 5. Vibe Coding（氛围编程）

**2025年AI跨越了关键能力阈值**——仅凭英语就能构建各种令人印象深刻的程序，完全忘记代码的存在。

Karpathy在X上随口创造了"vibe coding"这个词，没想到传播如此之广：

> "与AI结对编程，不要深究，只是vibe。接受一切，保持流动状态。" ——@karpathy

- **编程不再是专业人士专属**：任何人都能做
- **专业人士也能写更多**：否则永远不会被写出来的软件
- Karpathy用Rust vibecoded了自定义BPE tokenizer，写了MenuGen、LLM Council等项目

Karpathy引用自己更早的文章《Power to the People》：与所有其他技术相反，普通人从LLM中获益远超专业人士、企业和政府。

### 6. Nano Banana / LLM GUI

Google Gemini Nano Banana是2025年最**令人难以置信的范式转变模型**之一。

Karpathy将LLM视为继70-80年代计算机之后的下一个主要计算范式，因此会看到类似的创新：

- **个人计算的等价物**
- **微控制器（认知核心）的等价物**
- **互联网（Agent之间）的等价物**

**GUI的等价物在哪里？**

- 当前"与LLM聊天"有点像80年代向计算机控制台输入命令
- 文本是计算机（和LLM）偏爱的原始数据表示，但**不是人类偏爱的输入格式**
- 人类实际上**讨厌阅读文本**——它缓慢且费力
- 相反，人类喜欢**视觉和空间化**地消费信息——这就是为什么传统计算中发明了GUI

**LLM应该用什么格式与我们交流？**

图像、信息图、幻灯片、白板、动画/视频、网页应用等。

Nano Banana是**LLM GUI的早期雏形**——不只是图像生成本身，而是文本生成、图像生成和世界知识的**联合能力**，全部纠缠在模型权重中。
