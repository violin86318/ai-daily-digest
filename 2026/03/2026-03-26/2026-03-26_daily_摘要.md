## 🔴 GitHub Copilot 政策更新：4月24日起默认用用户交互数据训练模型

**来源**: GitHub Blog | [原文](https://github.blog/news-insights/company-news/updates-to-github-copilot-interaction-data-usage-policy/)

GitHub 宣布重大政策变更：**2026年4月24日起**，Copilot Free、Pro、Pro+ 用户的交互数据（输入、输出、代码片段及相关上下文）将默认用于训练和改进 AI 模型，用户需主动退出才能避免。

**核心内容**：
- 收集范围：接受的输出、发送的输入、游标位置的代码上下文、编写的注释和文档、文件名和仓库结构、Copilot 功能交互（聊天、内联建议等）、用户反馈
- **不收集**：Copilot Business/Enterprise 用户数据、退出的用户数据、私有仓库"静止状态"内容（issues、discussions 等）
- 数据可与 Microsoft 等关联公司共享，但不会给第三方 AI 模型提供商

**影响**：Copilot Business/Enterprise 不受影响。个人用户如不想参与，需在 4 月 24 日前在 Settings → Copilot → Privacy 中退出。

**评论精选**：
> "This aligns with established industry practices"——但实际上这是默认 opt-in，而非 opt-out，引发隐私倡导者强烈批评。

## 🤖 ARC-AGI-3 发布：首个交互式 AI 推理基准测试

**来源**: ARC Prize | [原文](https://arcprize.org/arc-agi/3)

ARC-AGI 团队发布 **ARC-AGI-3**，这是首个交互式推理基准测试，挑战 AI 代理在新环境中探索、动态获取目标、构建适应性世界模型并进行持续学习。

**核心特点**：
- 100% 人类可解决的交互式环境
- 测试技能获取效率、长期规划和稀疏反馈下的适应能力
- 重视经验驱动的多步骤适应
- 提供可重放的运行记录和开发者工具包

**设计原则**：人类可快速上手、无预加载知识或隐藏提示、清晰目标+有意义反馈、防止暴力记忆化的新奇设计

**意义**：100% 分数意味着 AI 代理能像人类一样高效地完成每个环境中的任务。只要 AI 与人类学习之间存在差距，AGI 就还未到来。

## 🧠 游戏开发者反思：AI 编程 agent 的七宗罪

**来源**: Mario Zechner Blog | [原文](https://mariozechner.at/posts/2026-03-25-thoughts-on-slowing-the-fuck-down/)

资深游戏开发者 Mario Zechner（libgdx、QUAKE III 引擎贡献者）发表深度反思，批评当前 AI 编程 agent 的使用方式：

**核心观点**：
1. **无瓶颈的代码烂摊子**：人类是瓶颈——人类每天能写的 bug 有限。Agent 军队没有瓶颈，错误会以指数级复合，最终形成无法恢复的复杂度灾难
2. **零学习能力**：Agent 不会从错误中学习，会一遍遍重复同样的错误
3. **本地视角导致碎片化**：每个 Agent 只看到代码库的一小部分决策，导致大量代码重复和无意义的抽象
4. **Agentic Search 召回率低**：代码库越大，Agent 越找不到所有需要改动的代码

**建议**：
- 让 Agent 做无聊的事（不重要的、临时的内部工具）
- 架构、API 等定义系统形态的核心部分必须手写
- 给自己设限（Agent 每天生成的代码量不超过你能真正 review 的量）
- 慢下来，给自己思考的时间

### 原始内容 1: GitHub Copilot 政策更新

从2026年4月24日起，GitHub Copilot Free、Pro和Pro+用户的交互数据——包括输入、输出、代码片段及相关上下文——将被用于训练和改进AI模型，除非用户选择退出。

GitHub在博文中表示，这一方法"与既定行业实践相符"，将提高所有用户的模型性能。通过参与该项目，用户将帮助模型更好地理解开发工作流程，提供更准确和安全的代码模式建议，并在问题进入生产环境之前帮助用户发现潜在bug。

收集的交互数据包括：用户接受或修改的输出、发送给GitHub Copilot的输入（包括显示给模型的代码片段）、游标位置的代码上下文、用户编写的注释和文档、文件名、仓库结构和导航模式、与Copilot功能的交互（聊天、内联建议等）以及用户的反馈（赞成/反对评级）。
