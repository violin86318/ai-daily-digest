## 🛡️ 工程与工具：MCP Context Mode 上下文消耗降低 98%

MCP 成为 AI 代理标准，但每个工具调用都会向 200K 上下文窗口填充数据。Playwright 快照消耗 56KB，三十个 GitHub issues 消耗 59KB。运行 30 分钟后，40% 上下文已耗尽。

**Context Mode** 作为 MCP 服务器位于 Claude Code 和输出之间：315KB → 5.4KB，98% reduction。

- **Sandbox 模式**：每个 execute 调用生成独立子进程，原始数据（日志、API 响应、快照）永不离开沙箱
- **知识库模式**：使用 SQLite FTS5 全文搜索，BM25 排名算法，按标题分块 markdown
- **效果验证**：Playwright 56KB → 299B，GitHub issues 20个 59KB → 1.1KB，会话时间从 30 分钟延长至 3 小时

## 🔬 AI 研究：阿里 Qwen3.5 本地运行达 Sonnet 4.5 水平

Alibaba 发布 Qwen3.5 Medium Model 系列，四款模型支持 agentic tool calling，其中三款开源（Apache 2.0）：

- **Qwen3.5-35B-A3B**：总参数 350亿，激活仅 3亿，MoE 层 256 专家
- **思考模式**：默认启用，模型在生成最终答案前内部推理（`<think>` 标签）
- **突破性**：32GB VRAM 消费级 GPU 即可实现 100万 token 上下文，4-bit 量化近乎无损

**基准测试**：35B-A3B 超越 Qwen3-235B（更大的模型），以及 GPT-5-mini 和 Claude Sonnet 4.5。

**API 价格**：输入 $0.1/M tokens，输出 $0.4/M tokens — Claude Sonnet 4.5 的 1/36。
