## 🛠️ 工程与工具：MCP 输出优化 98%

Context Mode 是一个 MCP 服务器， sits between Claude Code and tool outputs。它通过两个核心机制实现压缩：沙箱模式（Sandbox）隔离执行环境，仅将 stdout 放入对话上下文；知识库模式（Knowledge Base）使用 SQLite FTS5 + BM25 排名算法索引文档。测试结果：Playwright 快照 56KB → 299B，GitHub issues 20 条 59KB → 1.1KB，500 行 CSV 85KB → 222B。安装方式：`/plugin marketplace add mksglu/claude-context-mode`

**来源：** [HackerNews](https://news.ycombinator.com/)

## 🔬 探索与研究：认知债务——当速度超过理解

一篇深度文章提出「认知债务」概念：当 AI 生成代码的速度超过工程师理解代码的速度时，债务便开始累积。与技术债务不同，认知债务对 velocity metrics 不可见，却在后台持续累积。三种失败模式：(1) 代码存在越久越危险，因为周围人的 context window 已关闭；(2) 3AM 告警时，工程师在调试一个「黑盒写的黑盒」；(3) 初级工程师永远无法形成架构判断的直觉。核心问题：组织无法优化它无法衡量的东西。

**来源：** [HackerNews](https://news.ycombinator.com/)

## 🌐 开源与工具：llmfit——找到你硬件能跑的 LLM

一个终端工具，可以根据系统 RAM、CPU、GPU 自动检测硬件并评分每个模型，支持质量、速度、fit、context 四个维度。内置交互式 TUI 和经典 CLI 模式，支持多 GPU、MoE 架构、动态量化选择、速度估算，以及 Ollama、llama.cpp、MLX 等本地运行时。安装：`curl -fsSL https://llmfit.axjns.dev/install.sh | sh` 或 `brew install llmfit`。
