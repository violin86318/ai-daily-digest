### 认知债务：当速度超过理解

## 🛠️ 工程与工具

### Context Mode：将 MCP 输出减少 98%
开发者构建了 MCP 服务器 Context Mode，可将 Claude Code 的上下文消耗减少 98%。原始输出 315 KB 压缩至 5.4 KB，使对话时长从 30 分钟延长至 3 小时。该方案包含两个核心机制：沙箱执行（隔离子进程，输出仅返回 stdout）和知识库索引（SQLite FTS5 + BM25 排名）。

**来源：** [Mert Köseoğlu Blog](http://mksg.lu/blog/context-mode)

### MCP 已死，CLI 万岁
开发者 Eric Holmes 断言 MCP 协议正在消亡，理由包括：1）LLM 本身擅长学习使用 CLI；2）CLI 对人类和机器同样可用，调试更透明；3）CLI 可组合（pipe、grep、jq）；4）MCP 初始化不稳定、重复认证、权限粒度不足。作者建议：优先构建 CLI，MCP 仅作为无替代方案时的最后选择。

**来源：** [Eric Holmes Blog](http://ejholmes.github.io/2026/02/28/mcp-is-dead-long-live-the-cli.html)

### WebMCP：Chrome 的 agentic web 方案
Chrome 发布 WebMCP 早期预览，提供两个 API：声明式 API（处理标准 HTML 表单操作）和命令式 API（处理复杂动态交互）。目标是将网站暴露为「agent-ready」状态，使 AI 代理能更可靠地执行订票、电商结账、客服工单等任务。

**来源：** [Chrome Developers Blog](https://developer.chrome.com/blog/webmcp-epp)

### Servo 浏览器一月进展
Servo 渲染引擎发布月度更新，包括预加载优化、表单增强、详情样式改进等。

## 💡 编者观察

本周最值得关注的趋势是 AI 开发工具的「反璞归真」。MCP 协议曾被视为连接 AI 与服务的标准，但开发者迅速意识到它带来了不必要的复杂性——LLM 本身擅长使用 CLI，而 CLI 的透明性、可调试性和组合性是 MCP 无法替代的。这与「认知债务」概念形成有趣呼应：工具越复杂，人类越难理解系统的运作方式。
