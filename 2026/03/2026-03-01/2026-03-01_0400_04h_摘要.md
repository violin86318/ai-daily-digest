## 🛡️ 工程与工具：Context Mode —— MCP 输出压缩 98%

MCP 工具每次调用都会向 Claude Code 的 20 万 token 上下文窗口倾泻原始数据。一个 Playwright 截图 56 KB，二十个 GitHub Issues 59 KB。运行测试、读取日志、抓取文档 —— 每个响应都在蚕食剩余空间。Cloudflare 展示了工具定义可以压缩 99.9%，但输出侧从未被解决。

Context Mode 是一个 MCP 服务器，坐在 Claude Code 和这些输出之间。315 KB 变成 5.4 KB，**98% reduction**。它通过两种机制工作：

**沙盒执行**：每个 `execute` 调用生成隔离的子进程，脚本无法访问彼此的内存或状态。子进程运行代码、捕获 stdout，只有 stdout 进入对话上下文。原始数据 —— 日志文件、API 响应、截图 —— 永远不离开沙盒。

**知识库索引**：`index` 工具按标题分块 markdown 内容，同时保持代码块完整，然后存储在 SQLite FTS5 虚拟表中。搜索使用 BM25 排名算法。

**实测数据：**
- Playwright 截图：56 KB → 299 B
- GitHub Issues (20条)：59 KB → 1.1 KB
- 访问日志 (500请求)：45 KB → 155 B
- 完整会话：315 KB → 5.4 KB，会话时间从 ~30 分钟延长到 ~3 小时

## 📚 认知债务：当速度超过理解

一篇深度博客文章提出「认知债务」（Cognitive Debt）概念，描述 AI 辅助开发中的新问题：

**核心问题**：AI 生成代码的速度远超人类理解代码的速度。当工程师手动写代码时，生产和吸收是耦合的 —— 输入的过程强制认知参与。AI 改变了这个等式：提示词几秒生成数百行，工程师只负责 review 和调整。输出加速了，但吸收无法同比例加速。

**三个失败模式：**
1. **信任反转**：传统上代码存在越久越可信；AI 代码恰恰相反，时间越长越危险，因为周围人的上下文窗口已经完全关闭
2. **深夜调试**：凌晨 3 点警铃响起值班工程师面对自己没构建的系统、AI 生成的工具、自己没有的文档——「你在调试一个黑盒写的黑盒」
3. **后继无人**：依赖 AI 的初级工程师永远无法形成手动实现带来的直觉和组织知识

**根本矛盾**：组织无法优化它无法测量的东西。速度是可量化的，理解不是。系统正在正确优化它测量的东西，但测量不再捕捉真正重要的东西。

**来源：** [Rockoder - Cognitive Debt](https://www.rockoder.com/beyondthecode/cognitive-debt-when-velocity-exceeds-comprehension/)
